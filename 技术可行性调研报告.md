## 一、北大教学网技术架构分析

### 1.1 平台基础

北大教学网 (`course.pku.edu.cn`) 基于 **Blackboard Learn** 构建，这是一款企业级学习管理系统 (LMS)。核心特点：

- **认证系统**：通过北大 IAAA 统一认证（OAuth 2.0 流程）
- **前端渲染**：服务端渲染 HTML，非 SPA 应用
- **数据获取**：HTML 页面解析 + 部分 REST API 混合模式

## 二、核心 API 端点详解

### 2.1 认证流程（IAAA OAuth）

```
用户输入账号密码
    ↓
POST https://iaaa.pku.edu.cn/iaaa/isMobileAuthen.do  // 检查是否需要 OTP
    ↓
POST https://iaaa.pku.edu.cn/iaaa/oauthlogin.do      // 获取 OAuth token
    ↓
GET  https://course.pku.edu.cn/webapps/bb-sso-BBLEARN/execute/authValidate/campusLogin?token=xxx
    ↓ (多次重定向)
Cookie 写入，登录完成
```

**关键参数**：
- `appid`: `"blackboard"` (教学网) 或 `"portalPublicQuery"` (门户)
- `redirUrl`: 对应的回调地址
- `otpCode`: 手机令牌码（可选，空串表示不提供）

**注意事项**：
- OTP 认证：部分账号启用了手机令牌，需要额外处理
- Cookie 会话：登录后的 session 可以持久化复用
- 错误码：`E05` = OTP 码错误，`E21` = 尝试过多需等待 30 分钟

### 2.2 REST API 端点（JSON 响应）

pku3b 已验证以下 Blackboard Learn REST API 可用：

```
# 用户信息
GET /learn/api/public/v1/users/me
→ { "id": "xxx" }

# 用户课程列表
GET /learn/api/public/v1/users/{user_id}/courses
→ { "results": [{ "courseId": "xxx", "courseRoleId": "Student" }] }

# 课程详情
GET /learn/api/public/v1/courses/{course_id}
→ { "name": "...", "description": "...", ... }

# 成绩册列
GET /learn/api/public/v2/courses/{course_id}/gradebook/columns
→ { "results": [{ "id": "xxx", "name": "...", "grading": {...}, "score": {...} }] }

# 成绩册数据
GET /learn/api/public/v2/courses/{course_id}/gradebook/columns/{column_id}/users
→ { "results": [{ "displayGrade": { "score": 85 } }] }
```

**重要发现**：REST API **没有**直接的 assignment/deadline 端点可用。作业 DDL 必须通过 HTML 解析获取。

### 2.3 HTML 解析端点（核心 DDL 来源）

```python
# 课程主页（获取课程菜单）
GET https://course.pku.edu.cn/webapps/blackboard/execute/announcement
    ?method=search&context=course_entry&course_id={key}&handle=announcements_entry&mode=view

# 课程内容列表（获取作业列表）
GET https://course.pku.edu.cn/webapps/blackboard/content/listContent.jsp
    ?content_id={id}&course_id={course_id}

# 作业上传页面（获取 DDL）← 核心
GET https://course.pku.edu.cn/webapps/assignment/uploadAssignment
    ?action=newAttempt&content_id={id}&course_id={course_id}

# 作业查看页面（获取提交状态）
GET https://course.pku.edu.cn/webapps/assignment/uploadAssignment
    ?mode=view&content_id={id}&course_id={course_id}
```

### 2.4 DDL 解析逻辑

作业 DDL 从上传页面的 `#assignMeta2 + div` 元素提取，格式为：

```
2026年6月20日 星期五 下午11:59
```

正则解析：
```regex
(\d{4})年(\d{1,2})月(\d{1,2})日 星期. (上午|下午)(\d{1,2}):(\d{1,2})
```

**边界情况**：
- 部分作业无截止时间（DDL 为 None）
- 部分作业的 DDL 文本格式不标准，正则可能匹配失败
- 需要处理 `上午/下午` 到 24 小时制的转换

## 三、技术路线方案

### 方案 A：浏览器扩展 + 后端服务（推荐）

```
┌─────────────────┐     ┌──────────────────┐     ┌─────────────┐
│  浏览器扩展      │────▶│  后端服务         │────▶│  Notion API │
│  (数据采集层)    │     │  (数据处理层)     │     │  (数据展示层)│
└─────────────────┘     └──────────────────┘     └─────────────┘
```

**架构**：

1. **浏览器扩展**（数据采集层）
   - 复用 pku3b 的认证逻辑，通过教学网页面内的 Cookie 发起请求
   - 定时扫描所有课程的作业列表
   - 解析 HTML 提取 DDL 数据
   - 推送到后端服务

2. **后端服务**（数据处理层）
   - 接收并存储 DDL 数据
   - 定时同步（可通过扩展触发或独立定时任务）
   - 数据清洗和标准化
   - 调用 Notion API 写入

3. **Notion 集成**（数据展示层）
   - 创建 DDL 数据库
   - 自动同步更新
   - 用户可自由编辑和自定义视图

**技术栈建议**：
- 浏览器扩展：TypeScript + Chrome Extension Manifest V3
- 后端：Node.js/Python + SQLite（轻量化）
- Notion：官方 API SDK

**优势**：
- 浏览器扩展可直接复用教学网的登录态，无需用户额外输入密码
- pku3b 的 HTML 解析逻辑可以直接移植
- 后端服务可以做增量同步，减少请求量

**劣势**：
- 需要用户安装浏览器扩展
- 后端服务需要部署和维护

---

### 方案 B：纯命令行工具 + 定时任务（轻量化）

```
┌─────────────────┐     ┌─────────────┐
│  CLI 工具        │────▶│  Notion API │
│  (pku3b 模式)    │     │             │
└─────────────────┘     └─────────────┘
        ↑
    定时执行 (cron)
```

**架构**：

直接参考 pku3b 的实现，构建一个专注于 DDL 同步的命令行工具：

1. 用户配置教学网账号密码（一次性）
2. 工具自动登录、遍历课程、提取 DDL
3. 通过 Notion API 写入数据库
4. 通过 cron/launchd 定时执行

**技术栈建议**：
- TypeScript/Python（开发效率优先）
- 复用 pku3b 的认证和解析逻辑
- Notion 官方 SDK

**优势**：
- 最轻量，无需后端服务
- pku3b 已验证全部核心逻辑
- 可以直接用 pku3b 作为依赖（Rust 二进制）或移植其逻辑

**劣势**：
- 需要用户在本地存储账号密码（或使用环境变量）
- 需要用户自行配置定时任务
- OTP 处理需要交互式输入

### 方案 C：教学网页面内嵌脚本（UserScript）

```
┌─────────────────────────────────┐
│  TamperMonkey 脚本              │
│  (在教学网页面内直接运行)        │
│  → 提取 DDL → 推送到 Notion     │
└─────────────────────────────────┘
```

**架构**：

类似 PKU-Art 的模式，通过 UserScript 在教学网页面内直接运行：

1. 脚本在教学网页面加载时自动执行
2. 利用页面已有的登录态和 DOM
3. 解析当前页面的作业信息
4. 通过 Notion API 直接推送

**优势**：
- 无需处理登录问题（直接复用页面 session）
- 安装最简单（只需 TamperMonkey）
- 可以利用 PKU-Art 的项目结构

**劣势**：
- 需要用户手动访问教学网才能触发同步
- 跨域请求可能受限（Notion API）
- 定时同步难以实现

### 4.3 Notion API 限制

- **速率限制**：平均 3 请求/秒（每秒级别）
- **分页**：每次最多返回 100 条结果
- **认证**：需要创建 Internal Integration 获取 API Token
- **权限**：需要手动将 Integration 添加到目标页面

### 5.2 焦虑值计算

```typescript
function calculateAnxiety(assignments: Assignment[]): number {
  const now = new Date();
  let anxiety = 0;

  for (const a of assignments) {
    if (a.status === '已完成') continue;

    const hoursLeft = (a.deadline.getTime() - now.getTime()) / (1000 * 60 * 60);

    if (hoursLeft < 0) {
      anxiety += 100; // 已过期，最高焦虑
    } else if (hoursLeft < 24) {
      anxiety += 80 * (1 - hoursLeft / 24); // 24小时内
    } else if (hoursLeft < 72) {
      anxiety += 40 * (1 - hoursLeft / 72); // 3天内
    } else {
      anxiety += 10; // 还有时间
    }
  }

  return Math.min(100, anxiety);
}
```

## 八、风险与注意事项

### 8.1 技术风险

| 风险 | 影响 | 缓解措施 |
|------|------|---------|
| 教学网页面结构变更 | HTML 解析失败 | 定期检查、多路径解析、异常告警 |
| IAAA 认证机制更新 | 登录失败 | 关注 pku3b 更新、预留适配时间 |

| OTP 认证 | 自动化受阻 | 支持手动输入、缓存 session |

### 8.2 合规风险

- **教学网使用条款**：确认自动化访问是否被允许
- **频率控制**：避免过于频繁的请求（建议间隔 > 30 分钟）
- **数据安全**：账号密码加密存储，不上传到第三方

### 8.3 用户体验

- **首次配置**：需要用户创建 Notion Integration，流程可能较复杂
- **OTP 处理**：启用了手机令牌的用户需要额外步骤
- **同步延迟**：Notion API 有速率限制，大量课程同步可能需要几分钟

### pku3b 认证流程（Rust → TypeScript 移植参考）

```typescript
// IAAA OAuth 登录
async function iaaaOAuthLogin(
  username: string,
  password: string,
  otpCode: string = ''
): Promise<string> {
  // 1. 检查是否需要 OTP
  const authCheck = await fetch(
    `https://iaaa.pku.edu.cn/iaaa/isMobileAuthen.do?appId=blackboard&userName=${username}`
  );
  const authData = await authCheck.json();

  // 2. 获取 OAuth token
  const loginRes = await fetch('https://iaaa.pku.edu.cn/iaaa/oauthlogin.do', {
    method: 'POST',
    headers: { 'Content-Type': 'application/x-www-form-urlencoded' },
    body: new URLSearchParams({
      appid: 'blackboard',
      userName: username,
      password: password,
      randCode: '',
      smsCode: '',
      otpCode: otpCode,
      redirUrl: 'http://course.pku.edu.cn/webapps/bb-sso-BBLEARN/execute/authValidate/campusLogin'
    })
  });
  const loginData = await loginRes.json();

  if (!loginData.success) {
    throw new Error(`Login failed: ${loginData.errors?.msg}`);
  }

  return loginData.token;
}
```

### DDL 解析逻辑（Rust → TypeScript 移植参考）

```typescript
function parseDeadline(text: string): Date | null {
  const regex = /(\d{4})年(\d{1,2})月(\d{1,2})日 星期. (上午|下午)(\d{1,2}):(\d{1,2})/;
  const match = text.match(regex);

  if (!match) return null;

  const [, year, month, day, period, hour, minute] = match;
  let h = parseInt(hour);

  if (period === '下午' && h < 12) {
    h += 12;
  }

  return new Date(
    parseInt(year),
    parseInt(month) - 1,
    parseInt(day),
    h,
    parseInt(minute)
  );
}
```
